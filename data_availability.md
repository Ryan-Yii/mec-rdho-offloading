# Data and Code Availability

This repository contains the source code, formal configurations, generated MEC
scenarios, raw experimental records, summary statistics, figures, and audit
metadata used for the RDHO study.

## Data Source

All experiments use simulated offline MEC task sets generated by
`src/task_generator.py`. No proprietary, confidential, human-subject, or
external measurement data are used. The generated tasks are deterministic for
the recorded `scenario_seed` values.

Representative task parameters and generation ranges are available in:

- `results/raw/task_parameters.csv`
- `results/raw/task_generation_ranges.csv`
- `paper_tables/task_parameters.md`
- `paper_tables/task_generation_ranges.md`

## Code and Configuration

- `src/`: system model, objective metrics, task generation, optimizers, and
  shared utilities.
- `experiments/`: formal runners, summaries, statistical tests, and figure
  generation.
- `configs/`: main, ablation, sensitivity, and scalability configurations.
- `tests/`: regression and methodology-contract tests.
- `docs/`: the methodology revision specification and TDD implementation plan.

The public repository is:

https://github.com/Ryan-Yii/mec-rdho-offloading

## Result Artefacts

- `results/raw/`: main, ablation, convergence, task, and scalability records.
- `results/sensitivity/raw/`: weight and penalty sensitivity records.
- `results/summary/`: descriptive summaries and paired Wilcoxon results.
- `results/figures/` and `results/sensitivity/figures/`: generated source plots.
- `paper_tables/`: Markdown exports of manuscript-facing tables.
- `figures/`: original manuscript-facing PNG copies.

Raw experiment rows expose `F0`, `F_search`, `F_report`, penalty decomposition,
normalized objective components, both random seeds, NFE use, and runtime. The
legacy `fitness` column is fixed to `reported_fitness`.

## Provenance and Audit

Run manifests under `results/manifests/` record the command, configuration
SHA-256 hash, source commit and branch, dirty-worktree state, timestamps,
environment versions, seed policy, NFE cap, and declared outputs. The separate
`postrun_analysis_manifest.json` records the committed analysis revision and
SHA-256 hashes for every source CSV/configuration and generated table/figure.

The complete pre-revision result set is preserved under
`results/legacy_before_methodology_revision/` and protected by
`legacy_snapshot_manifest.json` file hashes. It is not used as revised formal
output.

During the sensitivity run, three wall-clock observations crossed confirmed
operating-system suspend intervals. The affected rows were rerun with identical
scenario, algorithm, and configuration seeds. All deterministic non-runtime
fields reproduced within `1e-12`. The preserved pre-repair CSV is stored under
`results/sensitivity/audit/`. The executable command, post-run selection rule,
source/output hashes, original timings, replacement timings, and affected keys
are recorded in `results/manifests/sensitivity_runtime_repair.json` and can be
replayed by `experiments/repair_sensitivity_runtime.py`.

## Reproduction

Install `requirements.txt`, run `python -m pytest -q`, and execute the formal
commands in `README.md`. Each experiment runner requires `--force` when formal
outputs already exist and verifies the legacy snapshot before execution.
`experiments/regenerate_analysis.py` rebuilds all summaries, statistical tests,
and figures from the committed raw CSV inputs without rerunning optimizers.

## Scope Limitations

The datasets represent generated offline task sets and simulated fixed MEC
configurations. They do not constitute real-world network traces, subjective
QoE measurements, dynamic online arrivals, failure/retransmission data, or
deployment measurements. The repository's QoE and freshness outputs should be
interpreted as model-based proxies within this simulation scope.
